Мне тут попалось судоку, которое я не мог решить без полного перебора вариантов. Когда было сделано несколько итераций, то стало понятно, что решения можно найти лишь перебирая все варианты. Нельзя было найти ни одной подсказки.

Вот оно:

```
|   | 7 |   |   |   |   | 4 |   | 5 |
|   |   |   | 6 | 8 | 9 |   |   |   |
|   |   |   |   |   | 4 |   |   |   |
| 9 |   | 5 |   | 6 |   |   |   | 7 |
|   | 2 |   |   |   | 7 |   | 3 |   |
| 6 | 3 |   |   |   |   |   |   |   |
| 3 |   | 1 |   |   |   |   |   |   |
|   |   |   | 3 |   |   | 8 |   |   |
| 4 |   |   | 8 |   |   | 6 |   |   |
```

Стало интересно: может быть где-то ошибка в поиске решения? Все же человек не машина, а следовательно ошибается чаще.

Что мы знаем про простейшие решения судоку?

[Исходники](https://github.com/RussianPenguin/sudoku "Исходники к статье")

<!--more-->  
Знаем мы несколько паттернов:

- В одной ячейке не может быть повторяющихся чисел
- В строке или столбце так же не может быть повторяющихся чисел

Есть еще одно правило, которое можно пронаблюдать на примере.

Формулируется следующим образом:

- Если в одной малой таблице некоторое число может быть только в одном столбце или одной строке, то эта строка (столбец) в других малых таблицах не может содержать это число

Непонятно. Да. Непонятно. Но всегда можно посмотреть на примере.

Разместим все возможные варианты цифры 5

```
|   | 7 |   |   |   |   | 4 |   | 5 |
| 5 | 5 |   | 6 | 8 | 9 |   |   |   |
| 5 | 5 |   | 5 | 5 | 4 |   |   |   |
| 9 |   | 5 |   | 6 |   |   |   | 7 |
|   | 2 |   | 5 | 5 | 7 | 5 | 3 |   |

| 5 | 5 |   | 3 | 5 | 5 | 8 | 5 |   |
| 4 | 5 |   | 8 | 5 | 5 | 6 | 5 |   |
```

Очевидно, что пятерки, которые помечены красным существовать не могут.

Что в итоге? Напишем программу, которая пользуется вышеприведенными аксиомами и увидим, что иногда без полного перепора судоку не решается. :)
